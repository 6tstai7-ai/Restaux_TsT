# Test Workflow For Codex

For implementation tasks:

1. Read `CLAUDE.md`.
2. Read all files in `AI/`.
3. Inspect the relevant implementation files.
4. Run `pnpm test` before editing.
5. Report the baseline result.
6. Write a short work plan.
7. Add a test plan immediately after the work plan using `AI/TESTS.md`.
8. The test plan must list:
   - risks
   - edge cases
   - expected behavior
   - failure cases
   - regression checks
9. Use `AI/VALIDATION.md` to define the approach and impacted files.
10. Implement with minimal file changes.
11. Run focused validation after editing.
12. Update `AI/DEBUG_LOG.md` if behavior, architecture, workflow, or project guidance changed.

Do not skip the pre-implementation test analysis. It is part of the work, not a final-report extra.
